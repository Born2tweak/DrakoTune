## Challenges applied

1. **Could a single listener still create a pass?** No under the new schema/model/gates; duplicate keys and independent counts are mandatory.
2. **Can unanimous original preference on clean audio pass do-no-harm?** No; equivalence/non-inferiority and artifact endpoints replace processed-preference logic.
3. **Can metric gain hide mud/sibilance harm?** No; hard safety/harm axes prevent an unqualified pass.
4. **Can a permissive GitHub license justify a model?** No; code, weights, data, runtime, and outputs are separately reviewed.
5. **Can public data be used for commercial training?** Not without purpose-specific authorization.
6. **Can the desktop stack be chosen before license review?** No; DT-51 blocks DT-71.
7. **Can a paper/model enter production via the watcher?** No; watcher emits proposals and sandbox remains offline.
8. **Can tuning use confirmatory failures then claim on the same set?** No; grouped frozen splits and replication are explicit.
9. **Can release rely on one positive study?** No; DT-90 replication and DT-91 cross-functional gate are required.
10. **Can roadmap momentum expand to general mixing?** No; DT-92 requires a new explicit decision/evidence epoch.

## Residual planning risks

- Forty-eight milestones are intentionally long-range; external terms, product evidence, and implementation discoveries will require versioned replanning.
- Four bounded lanes improve throughput, but leases, resource quotas, immutable handoffs, and confirmatory-data isolation must be implemented before parallel mutation begins.
- Legal obligations cannot be resolved by this technical audit; DT-51/88/91 retain counsel gates.
- Meaningful perceptual/performance thresholds remain deliberately unset until their threshold-setting evidence exists.
- Product discovery could invalidate the isolated-vocal desktop hypothesis. If so, DT-53 updates the roadmap before substantial UI/data spend.

## Go/hold

**Go:** accept the package and begin DT-45 only.  
**Hold:** DSP feature expansion, model adoption, recruitment/spending, restricted acquisition, public binaries, quality claims, and general-mixing scope.
