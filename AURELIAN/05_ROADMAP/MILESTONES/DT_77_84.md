# H4 — Quality Improvement and Renewal (DT-77–DT-84)

## DT-77 — Evidence-Led Failure Taxonomy and Improvement Brief

| Field | Contract |
|---|---|
| 1. Stable ID and title | `DT-77 Evidence-Led Failure Taxonomy and Improvement Brief` |
| 2. Status | Blocked on DT-64, DT-66, DT-67, and DT-76 |
| 3. Objective | Convert observed calibration/listening/usability failures into ranked, falsifiable improvement candidates. |
| 4. User/research outcome | Engineering targets real harms and abstention gaps instead of speculative features. |
| 5. Why now | Valid evidence now exists; tuning earlier would contaminate it and optimize synthetic proxies. |
| 6. Evidence/risk/decision | R-001–R-004, R-016; D-005/D-008. |
| 7. In scope | Cluster failures by defect/stratum/stage/harm, root-cause hypotheses, comparator, value/cost/risk, experiments. |
| 8. Out of scope | Code changes, confirmatory data retuning, model adoption. |
| 9. Preconditions/dependencies | Complete calibration, confirmatory, and usability result packages. |
| 10. Affected locations | Failure registry, experiment briefs, risks/negative results, roadmap candidates. |
| 11. Data/license dependencies | Use data only within tuning-analysis grants; confirmatory raw access controlled. |
| 12. Architecture/API/schema | Failure IDs link inputs/slices/results/processor/UX and proposed experiment. |
| 13. Acceptance criteria | Every priority cites prevalence/severity/uncertainty/user impact and a disconfirming test; no proxy-only ranking. |
| 14. Automated verification | Trace/link/rights/completeness checks and reproducible aggregate generation. |
| 15. Manual/expert/legal/owner gates | **Automatic:** audio, evaluation, product-fit, privacy/data, prevalence, severity, and evidence-trace priority checks. **Human-only:** changing product scope, data rights, or public claims. |
| 16. Failure/adversarial tests | Loud but rare issue, average hides subgroup, rights-restricted example, metric/listening conflict, causal overreach. |
| 17. Performance/resource budget | Candidate cost/resource impact estimated; no implementation budget approved. |
| 18. Security/privacy | Examples redacted/authorized; no reidentification or public audio. |
| 19. Rollback/quarantine | Weak/contradictory hypotheses remain exploratory; no automatic roadmap promotion. |
| 20. Documentation/state updates | Negative results, evidence/risks, prioritized experiment briefs, decisions. |
| 21. Claim impact | May narrow current claim/abstention; cannot expand. |
| 22. Explicit non-authorization | Does not authorize feature/model implementation or use confirmatory set for tuning. |
| 23. Completion evidence | Reviewed taxonomy/priority matrix with traceable evidence and experiment candidates. |
| 24. Next-unblocked handoff | DT-78 preregisters the selected deterministic improvement; DT-81 selects research candidate. |
