|---|---|

| 4. User/research outcome | Participants can complete unbiased trials; researchers receive complete auditable data. |
| 5. Why now | Confirmatory evidence cannot be collected on exploratory append-only tooling. |

| 8. Out of scope | Recruitment, outcome analysis, changing locked protocol. |
| 9. Preconditions/dependencies | Immutable schema and timestamped confirmatory design. |
| 10. Affected locations | Listening web/UI/server/storage/tests/deployment runbook; evidence export. |
| 11. Data/license dependencies | Synthetic/authorized dry-run stimuli; framework/dependency licenses reviewed. |
| 12. Architecture/API/schema | Protocol-locked session API, one active response, append correction, audit events, redacted export. |
| 13. Acceptance criteria | No duplicate/lost responses under concurrency/retry; balance verified; WCAG-oriented task audit; replay identical. |
| 14. Automated verification | Load/concurrency/retry/offline/resume, side/order, treatment leak, schema, corruption, accessibility checks. |
| 15. Manual/expert/legal/owner gates | **Automatic:** security/privacy, accessibility, audio-playback, concurrency, assignment, and full synthetic dry-run checks. **Human-only:** opening access to real participants or changing consent/privacy commitments. |
| 16. Failure/adversarial tests | Double submit, network drop, clock skew, storage full, malicious filename, assignment disclosure, assistive-tech path. |
| 17. Performance/resource budget | Trial/audio load, response durability, concurrent participant, monitoring budgets from design and load test. |
| 18. Security/privacy | Least privilege, pseudonyms, consent gate, signed/authorized audio, retention, protected raw export. |
| 19. Rollback/quarantine | Disable study on integrity failure; partial sessions marked, not silently resumed into changed protocol. |
| 20. Documentation/state updates | Operations/incident/accessibility runbooks, threat model, platform validation report. |
| 21. Claim impact | Infrastructure only; no study/claim eligibility until valid execution. |
| 22. Explicit non-authorization | Does not open recruitment or public access. |
| 23. Completion evidence | Load/security/accessibility/dry-run reports, reproducible export/replay, approvals. |
| 24. Next-unblocked handoff | DT-66 and DT-67 can use the platform when their data/budget gates pass. |
