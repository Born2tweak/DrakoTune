# I — Desktop UX and Project Model

**Decision:** define the smallest coherent local-first desktop workflow.

Research non-destructive audio project patterns, local import/analyze/audition/adjust/export flows, progress/cancellation, crash recovery, accessibility, offline behavior, provenance display, A/B loudness matching, and transparent limitations. Validate whether isolated mono vocal is sufficient or context playback is required.

**Exclusions:** full DAW, plugin ecosystem, account/cloud sync, or generative features before the vertical slice.

**Deliverable:** states, project schema, error model, user-visible evidence, and usability study tasks.

**Stop when:** every action has inputs, outputs, persistence, cancellation, recovery, and acceptance evidence.
