# ADR 0002 — Aurelian integration: reference-only (no scaffold install)

- **Status:** accepted (2026-07-10)
- **Context:** the Aurelian OS (`Born2tweak/Aurelian`) offers a full project scaffold (`AGENTS.md`, `.aurelian/`, docs/research/reports templates). DrakoTune already has functioning equivalents: constitution (`docs/04-ai-rules.md` + milestone guardrails), canonical roadmap (`CURRENT_MILESTONE.md` + M00 audit reconciliation + `docs/implementation/DRAKOTUNE_MASTER_EXECUTION_ROADMAP.md`), decision records (`docs/decisions/`), evidence discipline (per-milestone completion reports, CI audio regression), research traceability (`docs/research/` + research-to-execution map). Aurelian's own install guide instructs merging with existing systems rather than duplicating.
- **Decision:** Aurelian is applied at the **operator level** (globally loaded by agents) and **referenced, not installed**, in this repo. No `.aurelian/`, no `AGENTS.md`, no second constitution or roadmap. Missing Aurelian-style artifacts were adopted as native DrakoTune docs: risk register (`docs/RISK_REGISTER.md`), validation plan (`docs/validation/`), dataset governance (`docs/data/`), research-to-execution map (`docs/implementation/`), research gaps (`docs/research/RESEARCH_GAPS.md`).
- **Consequences:** one source of truth per function; no drift between parallel systems. Agents follow Aurelian behavioral rules (read-before-edit, evidence-before-claims, narrowest-verification-first) via their global configuration; the repo enforces the project-specific versions through its existing guardrails.
- **Superseded if:** the project onboards agents that cannot receive global Aurelian rules — then embed the minimal kernel into a root instructions file (merge, don't overwrite).
