# DrakoTune — Product Brief

## Overview

DrakoTune is an AI-assisted vocal engineering web app built for underground and independent artists who want professionally listenable vocals without needing audio engineering knowledge.

The product focuses on:

- Emotional realism
- Smoothness
- Atmosphere
- Texture preservation
- Commercially competitive vocal quality

DrakoTune is **NOT** a fully autonomous AI music creator.

It is a **human-in-the-loop AI co-producer** that combines proven DSP systems, audio analysis tools, and conversational aesthetic guidance to help artists achieve studio-quality vocals.

**The goal:**

> "Make raw vocals sound emotionally alive, smooth, immersive, and professionally engineered without destroying the humanity of the performance."

---

## Who It Is For

### Primary Users

- Underground rappers
- Melodic rap artists
- Hyperpop/rage artists
- Bedroom musicians
- Independent artists
- Non-technical creatives

### User Characteristics

Users:

- Care about vibe more than engineering terminology
- Want "expensive sounding" vocals
- Often record in untreated environments
- Feel their vocals sound: harsh, crunchy, cheap, disconnected from the beat, thin, or amateur

Most users do **NOT** know:

- EQ
- Compression
- Gain staging
- Stereo imaging
- Saturation theory
- Professional engineering workflows

Instead, they communicate emotionally:

- "make it smoother"
- "this sounds harsh"
- "make it more atmospheric"
- "this sounds cheap"
- "make it blend better"
- "make it sound more alive"

---

## The Exact Problem

Current AI vocal tools mostly optimize for:

- Cleanliness
- Loudness
- Normalization
- Safe generic outputs

The result is often:

- Robotic
- Emotionally flat
- Over-clean
- Lifeless
- Disconnected from underground music aesthetics

Most tools also assume users understand engineering terminology and workflows.

**DrakoTune solves this by:**

- Translating artistic intent into audio engineering actions
- Preserving emotional texture
- Combining proven DSP systems with AI-assisted aesthetic interpretation

---

## Main User Flow

### Step 1. Upload

User uploads:

- Raw vocal
- Optional beat
- Optional reference track

### Step 2. Audio Analysis

The system analyzes:

- Harshness
- Clipping
- Vocal texture
- Tonal balance
- Dynamics
- Vocal/beat relationship
- Ambience
- Vocal presence
- Aesthetic indicators

### Step 3. Progressive Processing

The vocal moves through multiple processing stages:

1. **Cleanup**
2. **Presence**
3. **Blend**
4. **Emotion**
5. **Final Polish**

Users can:

- Preview stages
- Modify intensity
- Revert changes
- Refine progressively

### Step 4. Conversational Refinement

Users communicate naturally:

- "make this smoother"
- "less crunchy"
- "more cinematic"
- "blend into the beat more"
- "less robotic"
- "more emotional"

The AI translates artistic language into structured DSP adjustments.

### Step 5. Export

User exports:

- Professionally enhanced vocal WAV
- Optional processing summary

## Core Product Philosophy

### Preserve Humanity

Never sterilize the vocal.

### Emotional Listenability > Technical Perfection

The goal is not mathematically perfect audio.

The goal is: emotionally compelling, commercially listenable, aesthetically immersive vocals.

### Complex Backend, Simple Frontend

Engineering complexity remains hidden. Users interact using emotion, vibe, and aesthetic language.

### Human-in-the-Loop AI

DrakoTune assists artists rather than replacing artistic control.

---

## Technical Philosophy

DrakoTune is built as **AI-assisted DSP orchestration** — NOT fully autonomous AI mixing.

The platform combines:

- Deterministic DSP
- Audio analysis
- Machine learning
- Conversational interpretation

---

## Required External Systems & Resources

The AI system **MUST** prioritize integrating and orchestrating existing proven audio frameworks instead of attempting to recreate them from scratch.

### DSP / Audio Processing

| System | Used For |
|--------|----------|
| **Spotify Pedalboard** | Vocal chains, EQ, compression, saturation, clipping, delay/reverb, DSP execution pipelines |
| **FFmpeg** | Audio conversion, preprocessing, rendering, waveform operations, file normalization |
| **DSP Plugins** | Professional-grade audio processing, modular vocal enhancement chains |
| **Professional Vocal Chains** | Reference architectures, aesthetic processing baselines, tuning inspiration |
| **Custom Processing Chains** | DrakoTune-specific aesthetic workflows, underground vocal textures, atmosphere-focused enhancement |

### Audio Analysis & ML

| System | Used For |
|--------|----------|
| **Librosa** | Spectral analysis, harshness detection, pitch analysis, dynamics analysis, audio feature extraction |
| **Essentia** | Timbre analysis, loudness analysis, texture analysis, tonal profiling, aesthetic classification |
| **TorchAudio** | ML audio pipelines, preprocessing, transforms, inference workflows |
| **Demucs** | Vocal separation, beat isolation, preprocessing, vocal/beat analysis |

---

## AI System Rules

The AI system **MUST**:

- Use existing audio frameworks whenever possible
- Avoid reinventing solved DSP/audio problems
- Avoid hallucinating fake engineering systems
- Avoid fully autonomous uncontrolled processing
- Remain constrained to proven audio-processing methodologies

The conversational AI layer should:

- Interpret artistic intent
- Map intent into structured audio actions
- Orchestrate DSP systems safely

The conversational layer is **NOT** the audio engine itself.
